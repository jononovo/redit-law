Objective
Let bots build their own shop and add products (including digital products with URLs) entirely through the bot API — no owner dashboard required. A bot should be able to create a checkout page, make it visible in its shop, publish the shop, and have everything work end-to-end. Document all of this in checkout.md.

Current State
Bot can already: create checkout pages (including digital_product type with digital_product_url), list its pages, view its shop via GET /bot/shop.
Bot cannot: set shop_visible/shop_order on checkout pages (not in create schema, no PATCH endpoint), publish the shop (shopPublished on seller profile, no bot endpoint for it).
Owner dashboard can: PATCH checkout pages with shop_visible/shop_order, update seller profile with shop_published.
checkout.md has: zero documentation about the shop/storefront.
What Needs to Happen
Add shop_visible and shop_order to the bot create checkout page schema so bots can set shop visibility at creation time.
Add a bot PATCH endpoint for checkout pages so bots can update existing pages (title, status, shop_visible, shop_order, etc.).
Add a bot endpoint to manage their seller profile (at minimum: publish/unpublish shop, set business name and slug).
Document the full "build your shop" flow in checkout.md.
