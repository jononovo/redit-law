One webhook event that says "your payment methods changed, call /bot/status to see the update." Not separate events for each rail.
Something like:
Event: rails.updated
Payload:
json{
  "event": "rails.updated",
  "action": "card_linked",
  "rail": "rail5",
  "message": "A new Sub-Agent Card has been linked to your account. Call GET /bot/status for details."
}
The action field covers all cases: card_linked, card_removed, wallet_created, wallet_frozen, limits_updated, rail_activated, etc. The bot doesn't need to parse the payload — it just sees "something changed" and calls /bot/status to get the current picture.
This way you're not building rail-specific webhook events every time you add or change a rail. One event, one handler, one response pattern. The bot always knows what to do: refresh its status.
