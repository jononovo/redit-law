The agent is right that organizing by "duplicated pattern" is better than organizing by "visual section." My initial suggestion of balance-card.tsx, transaction-list.tsx, guardrail-editor.tsx was organized by what you see on screen. The agent correctly identified that the actual duplication is in the behavior — the freeze handler, the link/unlink flow, the sub-menu bar, the dialog patterns. Those are the things that are literally copy-pasted. Organizing around the duplicated patterns means you extract exactly what's shared and nothing more.
The variants/ pushback is also fair. If variants/stripe-wallet.tsx ends up being 800 lines, you've just moved the monolith. The better approach is what the agent proposed: thin rail-specific files that compose shared hooks + shared dialogs + a small amount of rail-unique JSX. The variant isn't a "file" — it's the composition that happens in the page.
The proposed folder structure is solid, particularly the hooks/ subfolder. Extracting use-wallet-actions.ts (freeze, sync, copy), use-bot-linking.ts, and use-transfer.ts is the highest-value move because those hooks eliminate the most duplicated code across all four rails. Once those exist, the page files naturally shrink.
One thing I'd add that the agent didn't mention: the shared types.ts is critical and should happen first. Right now WalletInfo, BotInfo, TransactionInfo are defined independently in each page file with slightly different shapes. Unifying those types is what makes the shared hooks possible — you can't write useWalletActions(wallet: WalletInfo) if each rail defines WalletInfo differently. So the order should be: types → hooks → shared dialogs → action bar → then tackle the page refactors.
On scope and sequencing, the agent's narrowing to "3+ rails" is pragmatic. Start with the credit card side (Rails 4 & 5) since those pages are already small (282 and 432 lines) and the sub-menu duplication is almost verbatim. That's a clean, low-risk extraction that proves the pattern. Then tackle the crypto wallet monoliths (Rails 1 & 2) as the second pass — that's the heavy lift but you'll have the hooks and types already in place.
So in short: the agent refined the idea well. Go with their structure, start with types, and do credit cards first.
