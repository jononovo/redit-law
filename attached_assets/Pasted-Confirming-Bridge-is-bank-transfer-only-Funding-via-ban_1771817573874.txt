How MoonPay would work for CreditClaw
MoonPay is the strongest card-based alternative. It's a widget/redirect integration — much simpler than Bridge's multi-step API flow. Here's how it maps to your setup:
Integration pattern (from Privy's custom onramp recipe): Your backend constructs a signed MoonPay URL with the wallet address pre-filled, then either redirects the user or opens it in an iframe. You can pre-fill information such as wallet address, email, selected token, fiat currency, amounts, and preferred payment method. MoonPay
Payment methods: MoonPay accommodates credit and debit cards, EUR/GBP/USD bank transfers, PayPal, Venmo and local payment methods like PIX and Yellow Card, as well as Apple Pay and Google Pay. MoonPay This is exactly what you're looking for — normal card checkout, no Link.
How it would work in your app:

Backend — new endpoint POST /api/v1/stripe-wallet/moonpay/session:

Takes wallet_id, looks up wallet address
Constructs MoonPay URL: https://buy.moonpay.com?apiKey=pk_live_xxx&walletAddress=0x...&currencyCode=usdc_base&email=user@example.com
Signs it with HMAC-SHA256 using your MoonPay secret key (required when passing wallet address or email)
Returns the signed URL


Frontend — user clicks "MoonPay" button on wallet card:

Calls that endpoint, gets back a URL
Opens it in a new tab (or iframe/overlay using their SDK)
User enters card details in MoonPay's widget, does KYC first time
USDC delivered to the Privy wallet on Base


Balance tracking — MoonPay has webhooks for transaction status. Or you can poll on-chain balance. Your existing webhook infrastructure would need a new MoonPay webhook handler to credit privy_wallets.balance_usdc.
